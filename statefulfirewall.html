Stateful Firewalls

Stateful packet filters were the next step in firewall technology. Firewalls of this type keep track of active connections by logging handshake messages. This means the firewalls must track the context and purpose of incoming packets to determine their the nature of the packets. For example, a stateful packet filter can determine if a packet is establishing a connection or transferring the bulk of the data, Stateless cannot do this since it only checks the header for IP addresses.[7]

Gupta et.al, show how a stateful packet filter can be implemented using the POX python library to create a controller for a Software Defined Networking tool that can detect fake packets sent from trusted sources. They tested this using the Ostinato Packet Builder tool to construct a fake packet. This proves to be a major improvement in terms of security than the stateless package filters, but they are less efficient in terms of throughput since this type of firewall needs to read more information from the package.[7]
